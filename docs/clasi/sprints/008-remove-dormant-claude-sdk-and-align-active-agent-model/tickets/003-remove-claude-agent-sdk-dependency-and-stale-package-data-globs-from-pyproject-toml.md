# Remove claude-agent-sdk dependency and stale package-data globs from pyproject.toml

## Description

After tickets 001 and 002 eliminate all `claude_agent_sdk` imports, `pyproject.toml` still
declares it as a runtime dependency. This ticket removes it and also cleans seven stale
`[tool.setuptools.package-data]` globs pointing to directories that do not exist under
`clasi/`:

```
"agents/**/*.md"
"agents/**/*.md.j2"
"agents/**/*.yaml"
"skills/*.md"
"instructions/*.md"
"instructions/languages/*.md"
"rules/*.md"
```

None of `clasi/agents/`, `clasi/skills/`, `clasi/instructions/`, or `clasi/rules/` exist
in the package tree — all active content lives under `clasi/plugin/`. The `"plugin/**/*"`
glob correctly covers all active content and is retained.

## Acceptance Criteria

- [x] `claude-agent-sdk` is absent from `[project.dependencies]` in `pyproject.toml`.
- [x] The seven stale globs listed above are removed from `[tool.setuptools.package-data]`.
- [x] `"plugin/**/*"` glob is still present and unchanged.
- [x] `uv lock` runs without error after the change.
- [x] `uv run pytest` passes; commit the updated `uv.lock`.

### Approach

1. Edit `pyproject.toml`: remove `"claude-agent-sdk>=0.1"` from `[project].dependencies`.
